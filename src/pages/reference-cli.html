Reference

CLI reference (overview)

Quick orientation for operators and contributors. Authoritative behavior lives in the shipped CLI help and waterfall-cli sources.

Global options

  • --spec-root, WATERFALL_SPEC_ROOT, .waterfall — spec resolution order is documented in the CLI README.
  • --backend — cline-ollama (default), cursor, claude-code.
  • --dry-run / --execute-agent — plan vs invoke.
  • --prompt-delivery embed|reference — inline workflow text vs absolute paths to shipped prompts.

Branch rules (typical)

cr start and story close on develop; lifecycle prompts, horizontals, sys start, and cr finish on feature/CR-… branches—see your spec’s CURSOR.md for normative policy.
