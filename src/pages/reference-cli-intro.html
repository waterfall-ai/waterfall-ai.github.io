What’s in the JSON

Top-level fields include commands, globalOptions, environment (rendered below as Environment variables), config (precedence, file locations, and config.notes such as lifecycle status) and dotWaterfall (both rendered under Configuration files), specResolution, plus name and title.

Each command row shows the full waterfall … invocation (from cliForm) with a Copy control; the dotted internal id sits on the right. Expand a row for the summary, a concrete Example, the required spec-repo Branch (from branchRequirement), and optional argvPrefix, positionals, commandOptions, promptFile, and orchestrationSteps in the JSON.

General examples

These are typical invocations; branch rules still apply (see each command’s Branch line below). Global flags such as --spec-root, --dry-run, and --execute-agent can go before the subcommand or after it (e.g. waterfall cr start --execute-agent "…"). The same switches are mirrored by WATERFALL_DRY_RUN and WATERFALL_EXECUTE_AGENT for containers and CI (see Global options and Environment variables).

  1. Usage and machine spec.

    waterfall help
waterfall help -o json

  2. Open a change request (spec repo on develop).

    waterfall cr start "OAuth login for partners"

  3. Lifecycle agent step on feature/CR-… with an inline hint (plan only).

    waterfall --dry-run cr to rq focus on session expiry edge cases

  4. Run the agent for a lifecycle step (real backend; set execute_agent=true in .waterfall or pass --execute-agent).

    waterfall cr to rq --execute-agent align RQ themes with checkout scope

  5. Close a story (spec repo on develop).

    waterfall story close STORY-042

  6. Finish the CR (checks + merge into develop; run from the CR feature branch).

    waterfall cr finish

The <hint> positional

Commands that take <hint> (or similar free text after fixed arguments) resolve operator hints in one of three ways—the same rules as in cli-help.json → operatorHint. Only one mode applies per invocation.

  • Inline (default). All remaining argv tokens after the fixed command words are joined with single spaces (shell quoting sets word boundaries). 
    waterfall rq to uc tighten validation around refunds
  • Stdin. Exactly one token - reads the entire hint from standard input as UTF-8 (good for multi-line text or scripts). 
    printf '%s\n' 'Line one
Line two' | waterfall uc to story -
  • File. Exactly one token starting with @ reads the file path after @; relative paths resolve against the current working directory. 
    waterfall test to story @notes/repro-hint.md

Branch policy: each command documents the required git branch in the spec repo (expanded row and branchRequirement in JSON). In short: cr start and story close on develop; lifecycle prompts, horizontals, sys start, and cr finish on feature/CR-… — see your spec’s CURSOR.md.
